## Features

<table>
<tr>
<td width="50%">

### Real-time messaging
WebSocket-powered with typing indicators, online/offline presence, read receipts, and instant delivery across channels, DMs, and threads.

### AI agents as teammates
Create agents via the admin panel. Mention `@agent` in any channel or open a dedicated 1-on-1 side panel. Responses stream token-by-token. Works with **any OpenAI-compatible API** — OpenAI, Anthropic, Ollama, vLLM, LiteLLM.

### Pluggable object storage
Local filesystem **or** any S3-compatible provider (AWS, IBM COS, Zadara, MinIO). Switch backends at runtime via admin panel with **zero-downtime file migration** and progress tracking.

</td>
<td width="50%">

### Threads, reactions, pins & custom emojis
Full conversation management with threaded replies, emoji reactions (custom + standard shortcodes), message pinning, and inline file previews with lightbox.

### Full-text search
PostgreSQL-native `tsvector` search with Portuguese stemming and unaccent normalization. Searches across messages, files, and users instantly.

### Progressive Web App (PWA)
Installable on desktop and mobile with standalone mode, offline fallback page, and dynamic theme-color that matches your selected theme.

</td>
</tr>
<tr>
<td width="50%">

### Admin panel
Live dashboard with stats, user management (roles, status, password reset), channel management (archive, read-only, retention), storage config, AI gateway setup, and RocketChat migration — all from the browser.

### 6 built-in themes
Dark Geo, Light Geo, Classic Dark, Dracula, Dark, and Light — with system-level theme-color integration for native OS feel.

</td>
<td width="50%">

### RocketChat migration
Full-fidelity import of users, channels, messages, reactions, threads, and files from RocketChat — with incremental sync and real-time progress logs. Both CLI and in-app migration engines.

### Message retention
Configurable auto-deletion policy per channel or globally, with an hourly background job. Set retention via admin panel, no restarts needed.

</td>
</tr>
</table>

## Quick Start

### Prerequisites

- **Go** 1.25+
- **Node.js** 20+
- **Docker** & Docker Compose

```bash
# 1. Start infrastructure (Postgres + Redis)
make dev

# 2. Set environment
export DATABASE_URL=postgres://glab:glab@localhost:5432/glab?sslmode=disable
export JWT_SECRET=dev-secret
export REDIS_URL=redis://localhost:6379

# 3. Apply database migrations
make migrate-up

# 4. Run backend
make backend

# 5. Run frontend (new terminal)
make frontend
```

Open **http://localhost:3000** and login with `admin` / `admin123`.

## Configuration

Glab uses **6 environment variables** for bootstrap. Everything else lives in the database and is managed via the **admin panel** at runtime — no restarts, no config files, no YAML.

### Bootstrap Environment Variables

| Variable | Default | Purpose |
|:---------|:--------|:--------|
| `DATABASE_URL` | *(required)* | PostgreSQL connection string |
| `JWT_SECRET` | *(required)* | JWT token signing secret |
| `REDIS_URL` | `redis://localhost:6379` | Redis connection string |
| `PORT` | `8080` | HTTP listen port |
| `CORS_ORIGIN` | `http://localhost:3000` | Allowed CORS origin |
| `UPLOAD_DIR` | `./uploads` | Local file storage directory |

### Admin Panel (`/admin`)

| Tab | What you can do |
|:----|:----------------|
| **Dashboard** | Live stats — users, channels, messages, files, storage usage, online count |
| **Users** | Create, edit, deactivate, change roles, reset passwords, set status |
| **Channels** | View, filter, sort, archive, toggle read-only, set per-channel retention |
| **Messages** | Global edit timeout configuration |
| **Storage** | Switch between local and S3-compatible storage. Test connection. Migrate files between backends with real-time progress |
| **AI** | Configure gateway URL, API token, and default model. Test connectivity. Any OpenAI-compatible endpoint |
| **Retention** | Global message retention policy with hourly cleanup job |
| **Migration** | Import everything from RocketChat — users, channels, messages, reactions, files |

## Deployment

Production runs on Docker Compose behind nginx with automatic SSL via Let's Encrypt.

```bash
# 1. Create deploy config (gitignored)
cat > .deploy.env << EOF
DEPLOY_HOST=your-server.com
DEPLOY_USER=root
DEPLOY_PORT=22
GLAB_DOMAIN=glab.example.com
EOF

# 2. Deploy (one command does everything)
./deploy.sh
```

The deploy script handles: file sync, nginx setup, SSL certificates, environment generation, Docker build, and container orchestration.

### Production Stack

| Service | Image | Purpose |
|:--------|:------|:--------|
| **glab-api** | Custom Go binary on Alpine | REST API + WebSocket hub + AI dispatcher |
| **glab-web** | Next.js standalone on Alpine | SSR frontend with PWA support |
| **glab-postgres** | `postgres:16-alpine` | Primary database with FTS |
| **glab-redis** | `redis:7-alpine` | Presence, typing, pub/sub fan-out |

## Architecture

```
                        ┌─────────────────────────────────────────┐
                        │              nginx (443)                 │
                        │       SSL termination + reverse proxy    │
                        ├────────────┬────────────┬───────────────┤
                        │  /api/*    │    /ws     │      /*       │
                        ▼            ▼            ▼               │
                ┌──────────────────────┐  ┌──────────────┐        │
                │    Go API (Chi)      │  │  Next.js 16  │        │
                │                      │  │              │        │
                │  REST      WebSocket │  │  React 19    │        │
                │  handlers    hub     │  │  Zustand     │        │
                │         │            │  │  shadcn/ui   │        │
                │    AI Dispatcher     │  │  Tailwind 4  │        │
                │     (SSE streaming)  │  │  PWA (SW)    │        │
                └──────┬───────┬───────┘  └──────────────┘        │
                       │       │                                  │
                  ┌────┴──┐ ┌──┴────┐                             │
                  │Postgres│ │ Redis │                             │
                  │  16    │ │   7   │                             │
                  │        │ │       │                             │
                  │ sqlc   │ │pub/sub│  ┌──────────────────┐      │
                  │ FTS    │ │presence│ │  Object Storage  │      │
                  │ UUIDs  │ │ cache │  │  Local / S3 /    │      │
                  └────────┘ └───────┘  │  IBM COS / MinIO │      │
                                        └──────────────────┘      │
                        └─────────────────────────────────────────┘
```

### Design Decisions

| Decision | Rationale |
|:---------|:----------|
| **sqlc** over ORM | Type-safe generated Go code, zero runtime reflection, full SQL control |
| **pgx COPY** for migration | Bulk inserts at max throughput — 100k+ messages/minute |
| **Redis pub/sub** for WebSocket | Enables horizontal scaling of API servers |
| **Pluggable storage** | Swap between local and S3-compatible at runtime via admin panel |
| **Config in DB** (not env vars) | AI gateway, storage, retention managed via admin panel — no restarts |
| **Agents as user accounts** | Agents appear naturally in channels, no special UI paths needed |
| **Manual service worker** | No `next-pwa` dependency — static `sw.js` for max compatibility |
| **AGPL-3.0 license** | Ensures improvements stay open-source |

## Project Structure

```
backend/                    Go API server
  cmd/glab/                 Entry point (routes, server, seed, retention job)
  internal/
    ai/                     AI gateway bridge + streaming dispatcher
    auth/                   JWT middleware + bcrypt password hashing
    config/                 Environment config (caarlos0/env)
    db/                     pgx connection pool
    handler/                HTTP handlers (REST + admin + RBAC)
    migration/              In-app RocketChat migration engine
    repository/             sqlc-generated type-safe queries
    retention/              Background message retention job
    storage/                Pluggable backends (local, S3) + migration engine
    ws/                     WebSocket hub + presence + typing
  migrations/               SQL migrations (golang-migrate)

frontend/                   Next.js 16 app (PWA-enabled)
  src/app/                  App router (login, chat, admin, settings, offline)
  src/components/           React components (shadcn/ui + custom)
  src/stores/               Zustand state management
  src/hooks/                WebSocket + keyboard shortcut hooks
  src/lib/                  API client, WebSocket client, types, emoji data

migrate/                    RocketChat migration CLI (separate Go module)
  cmd/migrate/              CLI entry point
  internal/
    rocketchat/             RC REST API client
    transform/              RC → Glab data transformation
    loader/                 Bulk insert via pgx COPY
```
